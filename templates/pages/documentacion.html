Documentación Técnica

Guía de integración, alcances y despliegue del ecosistema OBJET-X v1.1

1. Visión y Objetivos del Proyecto

Visión

Convertirse en una terminal inteligente de punto de venta ágil, capaz de automatizar el proceso de facturación y cobro mediante visión computacional, eliminando la necesidad de lectores de códigos de barras tradicionales y agilizando las auditorías de inventario en tiempo real.

Objetivos Actuales (Fase 1)

  • Reconocimiento de Bebidas: Clasificar de forma precisa diferentes tipos de bebidas mediante el modelo personalizado de YOLOv8.
  • Sincronización de Precios: Cruzar el identificador del modelo dinámicamente con un archivo local indexado para retornar nombres comerciales y costos monetarios al segundo.
  • Estructura Descentralizada: Permitir la gestión de la base de datos de manera externa (vía CSV/Excel) sin alterar una sola línea de código del pipeline de IA.

2. Requisitos Previos

Asegúrate de contar con las siguientes dependencias instaladas en tu entorno local antes de iniciar el servidor:

# Instalar librerías principales

pip install flask ultralytics pandas opencv-python

3. Guía de Ejecución Paso a Paso

1

Preparación del Dataset y Modelo

Las imágenes de bebidas etiquetadas en Roboflow son procesadas por el script entrenar.py. El resultado optimizado se guarda automáticamente como best.pt en la raíz del proyecto.

2

Configuración del Inventario (CSV)

Registra tus productos en el archivo productos.csv. Asegúrate de que el campo id_modelo coincida exactamente con la etiqueta que predice tu red neuronal.

3

Lanzamiento del Motor Flask

Ejecuta el servidor asíncrono desde tu consola de comandos local:

python app.py
4

Apertura del Panel e Interfaz

Abre tu navegador de preferencia e ingresa a la dirección http://localhost:5000. Haz clic en el botón de "Probar ahora" para activar la ventana modal del escáner continuo.

4. Estructura de la Base de Datos

El sistema consume los datos comerciales desde un plano estructurado en memoria. El formato de mapeo obligatorio para cada fila del inventario es el siguiente:

Campo Tipo Descripción Ejemplo
id_modelo String Clase exacta detectada por el archivo .pt Agua
nombre String Nombre público en la boleta/carrito Agua Mineral 500ml
categoria String Clasificación del rubro comercial Bebidas
precio Float Valor del costo unitario monetario 1.20
